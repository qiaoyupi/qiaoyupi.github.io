# 同一个文件二次上传无效的问题

解析

无效在于没有触发chang事件，没有触发change事件是因为你input里面储存的value值没有改变

# 解决方案







 


handleChange (e) {
  const arr = []
  for (const i of e.target.files) {
    arr.push(Object.freeze(i))
  }
  this.$emit('input', arr)
  e.target.value = ''
}

  1. 【推荐】关键点在e.target.value = ''，因为同一个文件第二次上传是不会触发change事件的，当input里储存的文件没有变化是，是不会触发的。所以上传之后把input的valu清空就可以了。
  2. 【不推荐】如果重新生成input标签，变成了新dom，自然input的value值也不存在了。
